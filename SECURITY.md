# SAGRI Security Architecture

## 1. Device Identity & Provisioning
Each Gateway and Field Node requires a unique device identity provisioned at the factory. 
- The `tools/provision.py` scripts generate the `partitions.csv` dynamically for NVS.
- Credentials (`wifi_pass`, `mqtt_pass`) are stored explicitly in the NVS partition rather than hard-coded in C headers. 
- During production, NVS partitions MUST be flashed with hardware encryption enabled.

## 2. Hardware Security Features (ESP32-S3 / H2)
The `sdkconfig.defaults` includes the flags strictly requiring Secure Boot and Flash Encryption:

### Secure Boot V2
```ini
CONFIG_SECURE_BOOT=y
CONFIG_SECURE_BOOT_V2_ENABLED=y
```
- Bootloader verifies the application partition using RSA-3072 signatures.
- Modifying `OTA_0` or `OTA_1` blocks externally causes the firmware to refuse to run.

### Flash Encryption
```ini
CONFIG_SECURE_FLASH_ENC_ENABLED=y
```
- AES-128-XTS encryption is enforced for the whole flash chip.
- The encryption key is randomly generated by the ESP32 hardware random number generator on first boot and locked inside eFuse Block 1. It is inaccessible via software or JTAG.

## 3. Communication Security

### Wi-Fi / MQTT (Gateway)
- **Wi-Fi**: WPA2/WPA3 PSK mode natively supported.
- **MQTT**: TLS 1.2+ is enforced. The embedded mbedTLS library uses pre-installed CA certificates to verify the server identity of the Cloud Broker (e.g. AWS IoT Core).

### Zigbee Network (Field Nodes)
- Configured as Zigbee 3.0 Standard Security.
- AES-128 Network Key distributed over the air securely using Zigbee Trust Center Link Key mechanisms during the joining phase.

## 4. OTA Firmware Delivery
The OTA update process maintains security across the entire chain:
1. Cloud server hosts `.bin` file on an HTTPS endpoint.
2. Gateway verifies TLS certificate of the server before downloading.
3. The image is saved to the standby application partition.
4. The ESP-IDF OTA framework automatically verifies the RSA-3072 signature appended to the compiled binary before activating the new partition marker.
5. If the new partition fails to initialize fully (Watchdog triggers), the system executes an automatic ROLLBACK to the previous partition.
