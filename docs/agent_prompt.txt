3. Functional Requirements
3.1  Sensing & Data Acquisition
Read temperature & humidity (DHT22 / SHT40) every configurable interval (default 30 s)
Read volumetric soil moisture at multiple depths (ADC, capacitive probe)
Read NPK soil nutrients via Modbus RTU over RS-485 UART
Pulse-count rain gauge via GPIO interrupt with debouncing
Read light intensity (BH1750, I2C) and CO2 ppm (SCD40, I2C)
Aggregate raw readings: median filter (window=5), calibration offset + gain stored in NVS

3.2  Actuation & Control
Drip irrigation valve control (GPIO relay + feedback current sensor)
Variable-speed water pump via PWM + RPM encoder feedback
LED grow-light PWM spectrum control (R/G/B/W channels, schedule-based)
Micro-ventilation fan with temperature setpoint PID controller
Fertigation injector dosing pump with flow-meter pulse counting

3.3  Zigbee Network
GW as Zigbee Coordinator — network formation, key distribution, permit join
Field Nodes as Zigbee End Devices — sleepy mode, poll-parent interval configurable
Custom Zigbee cluster: AGRI_CLUSTER (0xFF00) with manufacturer-specific attributes
Standard clusters: Basic, Power Config, Temperature, Humidity, Occupancy, On/Off, Level Control
Over-the-air Zigbee OTA cluster (cluster 0x0019) for field node firmware updates
Network health monitor — RSSI, LQI, parent change tracking, orphan detection

3.4  Gateway Services
Wi-Fi STA connection with automatic reconnect + exponential backoff
MQTT 3.1.1 over TLS 1.3 to cloud broker (configurable host, port, creds via NVS)
JSON telemetry publish: topic agri/{farm_id}/{node_id}/telemetry
JSON command subscribe: topic agri/{farm_id}/{node_id}/cmd
Local HTTP REST API on port 8080 — endpoints: /api/v1/nodes, /api/v1/sensors, /api/v1/actuators, /api/v1/config
Time sync via SNTP — all timestamps in ISO-8601 UTC
Edge buffering: store telemetry to SPIFFS ring-buffer when cloud offline (max 72 h data)

3.5  OTA Update System
GW: dual-partition OTA (A/B scheme) via HTTPS from update server
Field nodes: Zigbee OTA cluster — GW acts as OTA upgrade server
Version validation: semantic versioning, rollback on 3 consecutive boot failures
Atomic flash write, SHA-256 image verification, RSA-2048 signature check

6. Code Standards & Rules
6.1  Mandatory Coding Rules
All C code must follow C11 standard — use -std=c11 in cmake
Every ESP-IDF API call must check esp_err_t return and log on ESP_ERROR_CHECK_WITHOUT_ABORT
FreeRTOS tasks must include task-watchdog registration and periodic feeds
All NVS reads must handle ESP_ERR_NVS_NOT_FOUND with sane defaults
GPIO and peripheral inits must be idempotent — safe to call on restart
No busy-wait loops — use event groups, semaphores, or task notifications
Heap allocations must be paired with ESP_HEAP_CAPS_DEFAULT + NULL check
All structs in agri_data_model must be packed and version-tagged
MQTT payloads must be < 4096 bytes — chunk large datasets across messages
No hard-coded credentials — read all secrets from NVS namespace agri_config

9.1  app_main() Pattern — Gateway
void app_main(void) {
    ESP_LOGI(TAG, "Smart-Agri GW v%s starting", FW_VERSION);
    ESP_ERROR_CHECK(nvs_flash_init());
    ESP_ERROR_CHECK(esp_netif_init());
    ESP_ERROR_CHECK(esp_event_loop_create_default());
 
    nvs_config_load();              // Load all runtime config from NVS
    system_monitor_init();          // Heap + task watchdog setup
    sntp_sync_start();              // NTP time sync
    wifi_manager_start();           // Wi-Fi STA + reconnect loop
    zigbee_coordinator_start();     // Zigbee network formation
    cloud_buffer_init();            // SPIFFS ring-buffer
    mqtt_client_start();            // TLS MQTT session
    rest_api_start();               // HTTP REST on :8080
    ota_manager_start();            // OTA check task
    ESP_LOGI(TAG, "All services started. Free heap: %lu B",
             esp_get_free_heap_size());
}

9.2  Zigbee Attribute Reporting — Field Node
static void report_soil_cluster(agri_sensor_data_t *d) {
    esp_zb_zcl_report_attr_cmd_t cmd = {
        .address_mode  = ESP_ZB_APS_ADDR_MODE_16_ENDP_PRESENT,
        .direction     = ESP_ZB_ZCL_CMD_DIRECTION_TO_CLI,
        .clusterID     = AGRI_CLUSTER_ID,
        .attributeID   = ATTR_SOIL_MOISTURE,
        .data.type     = ESP_ZB_ZCL_ATTR_TYPE_U16,
        .data.value    = &d->soil_moisture_pct,
    };
    esp_zb_zcl_report_attr_cmd_req(&cmd);
}
